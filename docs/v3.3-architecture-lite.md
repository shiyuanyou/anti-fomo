## 关键技术决策

### 1. 后端框架：FastAPI
- **原因**：现代、类型安全、自动API文档、Pydantic数据校验
- **部署**：`uvicorn api.main:app --host 0.0.0.0 --port 8000`
- **进程管理**：使用systemd或pm2确保服务常驻

### 2. ORM：SQLAlchemy + Pydantic
- **模型定义**：SQLAlchemy模型与Pydantic模型分离（遵循FastAPI最佳实践）
- **数据库迁移**：使用Alembic（轻量，后续可升级）
- **SQLite位置**：项目根目录 `anti-fomo.db`，单文件便于备份和迁移

### 3. 前端构建配置
**Vite环境变量**：
```typescript
// .env.local
VITE_API_BASE_URL=/api  // 本地开发用proxy

// .env.cloud
VITE_API_BASE_URL=https://anti-fomo.yoyoostone.cn/api  // 云部署用绝对路径
```

**CORS配置**（FastAPI）：
```python
# api/main.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://anti-fomo.yoyoostone.cn"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 4. Nginx配置模板
```nginx
server {
    listen 80;
    server_name anti-fomo.yoyoostone.cn;
    
    # SSL证书配置（Certbot自动生成）
    listen 443 ssl http2;
    ssl_certificate /etc/letsencrypt/live/anti-fomo.yoyoostone.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/anti-fomo.yoyoostone.cn/privkey.pem;

    # 静态前端
    location / {
        root /home/user/anti-fomo/web/dist;
        try_files $uri $uri/ /index.html;
    }

    # API反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

# HTTP重定向到HTTPS
server {
    listen 80;
    server_name anti-fomo.yoyoostone.cn;
    return 301 https://$server_name$request_uri;
}
```

## 数据模型设计

### templates表（官方模板库）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(20) | PRIMARY KEY (如: "AF-T001") |
| name | VARCHAR(100) | 模板名称 |
| description | TEXT | 详细描述 |
| allocation | JSON | 资产配置 { "SPY": 60, "TLT": 40 } |
| metrics | JSON | 回测指标 { "expected_return": 0.08, "volatility": 0.12 } |
| tags | JSON | 标签数组 ["全天候", "保守型"] |

### shares表（用户分享）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(30) | PRIMARY KEY (如: "AF-SHARE-A1B2C3") |
| name | VARCHAR(100) | 用户自定义名称 |
| author | VARCHAR(50) | 作者名称 |
| config_json | JSON | 完整配置数据 |
| created_at | TIMESTAMP | 创建时间 |

## 迁移与部署流程

### 1. 模板数据迁移（一次性）
```bash
# 执行迁移脚本
python scripts/migrate_templates.py

# 该脚本将：
# 1. 连接SQLite数据库（自动创建表）
# 2. 读取 src/template_engine/templates.py
# 3. 插入到 templates 表
```

### 2. 服务器部署步骤
```bash
# 1. 上传代码到服务器
cd /home/user/anti-fomo

# 2. 安装后端依赖
pip install fastapi uvicorn sqlalchemy alembic

# 3. 初始化数据库
python scripts/migrate_templates.py

# 4. 构建前端
cd web && npm run build:cloud

# 5. 配置Nginx（复制配置模板并修改路径）
sudo cp deploy/nginx.conf /etc/nginx/sites-available/anti-fomo
sudo ln -s /etc/nginx/sites-available/anti-fomo /etc/nginx/sites-enabled/

# 6. 配置SSL（自动）
sudo certbot --nginx -d anti-fomo.yoyoostone.cn

# 7. 启动后端服务（systemd）
sudo systemctl enable anti-fomo-api
sudo systemctl start anti-fomo-api
```

### 3. systemd服务配置（后端常驻）
```ini
# /etc/systemd/system/anti-fomo-api.service
[Unit]
Description=Anti-FOMO FastAPI Backend
After=network.target

[Service]
User=user
WorkingDirectory=/home/user/anti-fomo
ExecStart=/usr/local/bin/uvicorn api.main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

## 性能与扩展考量

### SQLite vs PostgreSQL决策
- **当前选择SQLite的理由**：
  - 模板读取：QPS < 100，SQLite完全胜任
  - 用户分享写入：低频操作（每天几次）
  - 单文件便于备份和迁移
  - 零配置，无需Docker

- **未来迁移触发条件**：
  - 日活跃用户 > 1000人
  - 需要多服务器负载均衡
  - 需要复杂查询（如全文搜索、社交功能）

- **迁移成本**：
  - 只需修改 `api/database.py` 中的数据库URL
  - 使用SQLAlchemy抽象，业务代码0改动

### 性能优化
- **前端**：模板列表可添加虚拟滚动（若 > 100个）
- **后端**：SQLite添加索引 `(tags)` 和 `(created_at)`
- **Nginx**：启用静态文件缓存（`expires 7d`）
