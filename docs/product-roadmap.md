# anti-fomo 产品路线图（2026）

## 1. 目标与产品线分层

anti-fomo 从「可用工具」升级为「产品级工具」，采用三层产品线并行演进：

- L1：官网简洁版（免登录）
  - 目标：低门槛传播、快速体验、建立认知
  - 发布形态：云端 Web（域名直达）
- L2：本地部署开源版（GitHub）
  - 目标：可复现、可扩展、可二次开发
  - 发布形态：源码 + 数据 + 脚本
- L3：本地桌面增强版（PC 安装包）
  - 目标：开箱即用 + 插件服务 + 长期商业化空间
  - 发布形态：安装包（macOS/Windows）+ 插件生态

---

## 2. 版本节奏与里程碑

## V1（4~6 周）：官网可发布版

目标：对外可访问，用户 3 分钟内理解并完成首次核心操作。

范围：

- Home Page（项目初衷、三大能力、社区入口、维护者信息）
- 指数展示页（查询/筛选 + 基础指标 + 简图预览）
- 模板对比基础能力（先做核心摘要，不做复杂 AI）
- 最小社区展示页（静态信息）

退出标准（DoD）：

- 可公网访问，移动端与桌面端可读
- 无登录即可完成「看指数 + 做对比」主流程
- 页面无阻断错误，主要 API 稳定可用

## V2（6~8 周）：本地开源稳定版

目标：形成高质量本地部署体验，建立开发者口碑。

范围：

- 完成指数数据预处理与聚合入库（P0）
- 增强指数统计（周/月/季/年）
- 购物车 → 组合保存
- 本地 AI 查询入口（实验开关）
- 完整 README 与初始化脚本

退出标准（DoD）：

- 新用户 30 分钟内可按文档跑通
- 可离线查看核心指标（不依赖实时外部 API）
- 核心脚本可重复执行、结果一致

## V3（8~12 周）：桌面增强试运营版

目标：形成「安装即用 + 可增值」能力闭环。

范围：

- 桌面客户端打包（优先 macOS，其次 Windows）
- 插件机制 v1（先支持展示类插件）
- 组合包分享/下载最小闭环
- 本地增强入口（AI 分析、策略扩展）

退出标准（DoD）：

- 安装包开箱即用，首启可看到内置数据
- 插件可安装、启停、卸载
- 组合包分享链路可闭环验证

---

## 3. 优先级与依赖关系

核心依赖：

1. 数据底座（P0）是所有分析页面和 AI 能力的基础。
2. 指标口径统一是前后端一致展示的前提。
3. 云端与本地能力要明确分层，避免互相拖累。

推荐执行顺序：

1. 先冻结 V1 范围（防止需求扩散）
2. 并行推进 P0 数据底座
3. 完成 V1 交付后再扩展 V2 本地能力
4. V2 稳定后启动 V3 打包与插件机制

---

## 4. 运营与增长目标（建议）

V1 目标：

- 官网周活访问数
- 首页到指数页跳转率
- 模板对比使用次数

V2 目标：

- GitHub Star / Fork 增速
- 本地部署成功率
- issue 响应时效

V3 目标：

- 安装包下载量
- 插件安装率
- 社区组合包分享数量

---

## 5. 风险与缓解

- 风险：功能过多导致发布延迟
  - 缓解：严格执行「版本冻结」，新增需求默认进入下一版本
- 风险：外部数据源波动影响一致性
  - 缓解：落库快照 + 缓存 + 回退机制
- 风险：桌面版维护成本高
  - 缓解：先做最小安装体验，再逐步开放插件能力
