# Anti-FOMO v1 -- 目标定义

## 产品定位

Anti-FOMO = 投资决策节奏控制系统 (Decision Pacing System)

核心价值: 通过量化市场估值状态和个人资产波动, 帮助用户在正确的时间做正确粒度的决策,
减少因信息过载和情绪波动导致的频繁操作.

Anti-FOMO 不是:
- 投研系统 (不做个股推荐)
- 量化交易系统 (不做自动交易)
- 投顾系统 (不做买卖建议)

Anti-FOMO 是:
- 状态判断器 -- 市场贵不贵? 我的资产偏不偏?
- 节奏控制器 -- 今天该不该看盘? 这周需不需要操作?
- 决策提醒器 -- 什么时候需要再平衡? 新资金该怎么投?

## 从 v0 到 v1 的演进

### v0 现状 (已完成)

个股/组合波动监控工具:
- 持仓配置 (Web UI treemap 可视化)
- 日级波动计算
- 阈值告警
- AI 情绪分析
- 多渠道通知

v0 的问题: 纯"事后反应型", 只在波动发生后告警, 缺乏"事前判断"能力.
用户仍需自行判断市场环境, 容易陷入 FOMO 决策循环.

### v1 目标 (本次重构)

在 v0 基础上新增环境判断和决策节奏控制:

    个人资产状态 (v0 已有)
           +
    市场估值状态 (v1 新增核心)
           |
           v
    决策节奏控制 (v1 新增核心)
           |
           v
    行为建议输出 (日度判断 + 周度报告)

## MVP 功能范围

### P0 -- 必须实现

#### 1. 市场估值引擎 (market_engine)

目标指数:
- 沪深300 (000300) -- 大盘蓝筹代表
- 中证500 (000905) -- 中盘成长代表
- 上证50 (000016) -- 超大盘代表

输出:
- 各指数 PE (TTM) 百分位
- 各指数 PB 百分位
- 综合估值评分
- 市场状态判断: 便宜 / 正常 / 过热

数据源: akshare stock_index_pe_lg / stock_index_pb_lg (legulegu)
- 提供 2005 年至今的历史日级 PE/PB 数据
- 百分位由系统自行计算

核心评分逻辑:

    单指数评分 = (PE_percentile + PB_percentile) / 2

    综合评分 = weighted_avg(各指数评分, 权重可配)
      默认权重: 沪深300 = 0.5, 中证500 = 0.3, 上证50 = 0.2

    市场状态:
      综合评分 > 80  -> 过热
      20 <= 评分 <= 80 -> 正常
      评分 < 20  -> 便宜

#### 2. 决策节奏控制器 (decision_engine)

融合两个信号源:
- 个人资产波动 (v0 portfolio_engine 提供)
- 市场估值状态 (v1 market_engine 提供)

输出三个核心判断:

    1. 是否需要看盘?
       if 资产波动 > 阈值 AND 市场状态 == 极端(过热/便宜):
           需要关注
       else:
           忽略市场

    2. 是否需要调整?
       if 市场便宜 AND 有可投资金:
           可考虑定投
       if 市场过热 AND 权益仓位过重:
           注意风险

    3. 是否需要再平衡?
       if 任意持仓 权重偏离目标 > 阈值(默认5%):
           建议再平衡

#### 3. 日度状态判断

每日运行 (可手动触发或定时), 输出简洁判断:

    [2026-02-11] Anti-FOMO 判断
      市场状态: 正常 (综合评分 52.3)
      资产波动: 平稳
      建议: 无需操作, 忽略市场

控制台输出, 简短直接, 不产生焦虑.

#### 4. 周度市场报告

每周五收盘后自动生成, 内容包含:

    一. 市场估值概览
       - 各指数 PE/PB 百分位 及 本周变化
       - 综合评分 及 趋势

    二. 资产状态摘要
       - 组合波动情况
       - 持仓偏离度

    三. 行为建议
       - 本周是否需要操作
       - 下周关注点

    四. AI 分析 (可选)
       - 综合市场和资产状态的定性分析

#### 5. 数据缓存

市场估值数据不需要高频更新, 引入简单缓存机制:
- 本地文件缓存 (JSON/pickle)
- 当日已获取的数据不重复请求
- 百分位计算结果可缓存复用

### P1 -- 第二阶段 (v1 完成后)

以下功能不在 MVP 范围内, 但架构需为其预留扩展点:

- 宏观环境摘要 (AI + akshare 宏观指标)
- 月度资金投入建议
- 再平衡详细方案
- 月报/季报/年报
- Web 仪表盘 (市场状态可视化)
- 邮件/微信推送

## MVP 成功标准

MVP 完成时, 系统能做到:

1. 运行 python run.py -- 输出今日市场状态判断和行为建议
2. 运行 python run.py schedule -- 每日定时生成判断, 每周五生成周报
3. 运行 python serve.py -- 保留现有资产配置 UI 功能

用户视角的核心体验:

    每天花 10 秒看一眼系统输出, 就知道今天该不该关注市场.
    每周花 3 分钟看一份报告, 就知道这周的市场环境和自己的资产状况.
    不再因为盘中波动焦虑, 不再因为新闻标题冲动操作.

## 不做的事情 (明确排除)

MVP 阶段明确不做:
- 十四五规划解析 / NLP 政策解析
- 复杂宏观模型
- 多国家/多市场数据
- 高频(分钟级/盘中)数据
- 推荐算法 / 个股推荐
- 自动交易执行
- 移动端应用
