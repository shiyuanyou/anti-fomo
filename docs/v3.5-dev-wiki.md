# Anti-FOMO v3.5 开发 Wiki

> 本文档是 v3.5 的开发记忆，专注于前端 Vue 3 工程化重构与配置分享功能的实现。
> v3 专注于真实历史指标数据层，v3.5 在此基础上提升前端开发体验并增加社交分享功能。

---

## 战略背景

在经历了 v2 的 Web UI 开发后，我们发现现有的 Vanilla JS（原生 JavaScript 直接操作 DOM）模式已经达到了维护的极限：
1. `app.js` 等文件耦合严重，近千行的代码难以继续扩展。
2. 缺乏现代化的构建工具，导致在服务器上使用 Nginx 部署公共服务时，难以将静态资源与本地的配置文件（`config.asset.yaml`）解耦。
3. 为了实现分享页，我们需要前端既能在**本地自用**时读写服务器配置，又能在**云端部署**时作为纯静态页面（读写本地浏览器缓存）。

因此，v3.5 的核心任务分为两步：
- **前置阶段**：将前端使用 **Vue 3 + TypeScript + Vite** 进行组件化重构，并实现**双轨运行模式**。
- **功能阶段**：基于重构后的前端和新增的轻量级后端（FastAPI + SQLite），实现配置页的生成与社交分享功能。

## 前置阶段：前端工程化与双模式重构 (Phase 0)

### 1. 核心目标：双模式架构 (Dual Mode)

通过 Vite 的环境变量，实现一套前端代码，两种编译输出，适应不同场景：

- **本地模式 (Local Mode)** - 对应 `.env.local`
  - **场景**：用户在本地机器运行 `python3 serve.py`。
  - **行为**：通过 API 向后端发起读写请求，数据的持久化落地在宿主机的 `config.asset.yaml` 中。
  - **构建**：`npm run build:local`

- **云端模式 (Cloud Mode)** - 对应 `.env.cloud`
  - **场景**：部署在公网服务器，通过 Nginx 提供公共服务。
  - **行为**：完全与宿主机配置脱钩。读取用户的浏览器 `localStorage` 作为数据源；保存也仅固化在本地浏览器。
  - **构建**：`npm run build:cloud`

### 2. 前端目录重构计划

原有的 `web/` 目录将被备份，新的 `web/` 将采用标准的 Vue 3 工程结构：

```text
web/
├── .env.local             # VITE_APP_MODE=local
├── .env.cloud             # VITE_APP_MODE=cloud
├── vite.config.ts         # 本地开发代理配置
├── package.json           # npm 脚本
└── src/
    ├── main.ts            # Vue 应用入口
    ├── App.vue            # 主界面骨架
    ├── store/             # 状态管理 (封装 Local/Cloud 双轨存储策略)
    │   └── configStore.ts 
    ├── services/          # API 封装层
    ├── views/             # 核心视图层 (Portfolio, Templates, Compare, Share)
    └── components/        # 可复用组件 (ChartRadar, AssetList, Modal 等)
```

### 3. Nginx 部署架构的简化

重构后，后端的 `serve.py` 或 Nginx 仅需要挂载编译后的 `web/dist/` 目录。
- Nginx 只需要 serve 静态的 `dist` 目录。
- 将 `/api/` 路由反向代理给后端的 Python 服务。
- 彻底解决了原先前端与本地配置文件强绑定的部署痛点。

## 功能阶段：配置分享功能 (Phase 1)

基于 Vue 3 良好的组件化基础，我们开始构建真正的分享功能。

### 1. 用户场景
1. **即时分享**：用户配置完成后，快速生成带有二维码的分享链接发到投资群。
2. **存档分享**：创建永久的配置记录，供他人参考学习。
3. **一键复制**：查看他人分享页时，可点击“复制此配置”加载到自己的本地/浏览器中。

### 2. 后端 API 设计 (FastAPI + SQLite)

考虑到分享的数据需要公网持久化，将在服务器本地新增一个极轻量级的后端服务：
- **存储**：SQLite 单文件数据库 (`database.db`)
- **API 端点**：
  - `POST /api/share`：接收配置 JSON，返回短分享 ID (如 `AF-A1B2-C3D4`)
  - `GET /api/share/:id`：根据 ID 获取具体的配置数据

### 3. 数据库表设计
```sql
CREATE TABLE shares (
    id TEXT PRIMARY KEY,           -- AF-XXXX-XXXX
    name TEXT NOT NULL,            -- 配置名称
    author TEXT DEFAULT '',        -- 作者
    description TEXT DEFAULT '',   -- 投资理念
    config_json TEXT NOT NULL,     -- 配置数据内容
    metrics_json TEXT,             -- 回测指标
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4. 前端分享视图 (`ShareView.vue`)
- 新增独立的分享页面路由：`/share/:id`
- 展示用户的资产饼图、关键指标卡片和投资理念描述。
- 提供“复制此配置”、“生成二维码”等社交交互按钮。

## 阶段开发计划

### 步骤一：Vue 3 前端基础设施重构 (预计 2 天)
1. 备份原 `web/` 目录至 `web-legacy/`。
2. 使用 Vite 初始化 Vue 3 + TS 模板。
3. 实现 `configStore.ts` 封装本地/云端的双环境存储逻辑。
4. 将原有的 UI 和图表逻辑（Chart.js、Treemap）逐步迁移为 Vue 组件。

### 步骤二：后端与部署适配 (预计 0.5 天)
1. 修改 `serve.py`，将其静态文件服务目录指向 `web/dist/`。
2. 完善云端部署时的 Nginx 配置文件示例。

### 步骤三：开发分享 API 与数据库 (预计 1 天)
1. 编写 FastAPI 存储服务与 SQLite 的 CRUD 逻辑。
2. 实现 `AF-XXXX-XXXX` 格式的短链接生成算法。

### 步骤四：完善前端分享交互 (预计 1 天)
1. 在 Vue 3 项目中开发分享模态框。
2. 开发独立路由的 `ShareView.vue` 页面。
3. 集成二维码生成库并完成端到端测试。

## 成功指标
1. **工程化达标**：完全去除老旧的 Vanilla JS 模式，Nginx 能够使用 `build:cloud` 产物做到一键解耦部署。
2. **分享可用性**：生成的分享链接可以被外部无障碍访问并清晰展示用户的配置逻辑。
3. **交互丝滑**：在分享页点击“复制此配置”后，能够无缝覆盖当前浏览器的本地缓存并跳转至编辑页。
