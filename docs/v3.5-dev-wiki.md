# Anti-FOMO v3.5 开发 Wiki

> 本文档是 v3.5 的开发记忆，专注于实现配置分享功能。
> v3 专注于真实历史指标数据层，v3.5 在此基础上增加社交分享功能。

---

## 战略背景

v3 解决了模板指标数据可信度问题，使用真实历史数据计算年化收益、波动率等关键指标。
v3.5 在此基础上，增加**配置分享功能**，让用户能够：

1. **生成可分享的配置展示页**（带二维码）
2. **保存和分享投资思路**（小群交流、社交媒体）
3. **加载他人配置**（学习借鉴）

核心价值：
- **社交裂变**：通过分享吸引新用户
- **社区交流**：投资群内分享配置思路
- **存档回顾**：保存自己的配置历史

---

## 产品设计

### 用户场景
1. **即时分享**：用户配置完成后，快速生成分享链接，发到投资群
2. **存档分享**：创建永久的配置记录，供他人参考学习
3. **模板分享**：分享自己的配置作为模板，供他人复制使用

### 分享页内容
1. **头部信息**：
   - 配置名称（用户自定义）
   - 作者/昵称（可选）
   - 创建时间
   - 分享ID（如 `AF-A1B2-C3D4`）

2. **核心可视化**：
   - 资产配置饼图（按类型/区域）
   - 关键指标卡片（波动率、预期年化收益、最大回撤、夏普比率）
   - 与经典模板对比雷达图（可选）

3. **投资理念**：
   - 用户填写的理念总结
   - 配置特点描述

4. **交互功能**：
   - "复制此配置"按钮（加载到自己的编辑器中）
   - 二维码（方便移动端分享）
   - 分享到社交媒体按钮

## 技术架构

### 整体架构
```
用户浏览器 → Nginx → Python FastAPI → SQLite数据库
```

### 组件说明

#### 1. 前端扩展（现有Web UI）
- **新增功能**：
  - "生成分享页"按钮（在配置概览面板）
  - 分享模态框（输入配置名称、投资理念）
  - 二维码生成（qrcode.js）
  - 分享页路由：`/share/:id`

#### 2. 后端服务（新增）
- **框架**：Python FastAPI（轻量级）
- **数据库**：SQLite（单文件，简单可靠）
- **存储位置**：自有服务器本地
- **API端点**：
  - `POST /api/share` - 存储配置，返回分享ID
  - `GET /api/share/:id` - 获取配置数据
  - `GET /share/:id` - 渲染完整分享页（HTML）

#### 3. 数据库设计（SQLite）
```sql
CREATE TABLE shares (
    id TEXT PRIMARY KEY,           -- AF-XXXX-XXXX
    name TEXT NOT NULL,            -- 配置名称
    author TEXT DEFAULT '',        -- 作者（可选）
    description TEXT DEFAULT '',   -- 投资理念
    config_json TEXT NOT NULL,     -- 配置数据（JSON字符串）
    metrics_json TEXT,             -- 计算指标（JSON字符串）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP,          -- 过期时间（30天后）
    view_count INTEGER DEFAULT 0   -- 查看次数
);
```

#### 4. ID生成策略
- **格式**：`AF-XXXX-XXXX`（如 `AF-A1B2-C3D4`）
- **长度**：8位核心字符 + 分隔符
- **字符集**：大写字母+数字（36个字符）
- **容量**：36^8 ≈ 2.8万亿组合
- **唯一性**：前端生成，服务端验证冲突

### 前端ID生成算法
```javascript
function generateShareId() {
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  let id = 'AF-';
  for (let i = 0; i < 8; i++) {
    if (i === 4) id += '-';
    id += chars[Math.floor(Math.random() * chars.length)];
  }
  return id; // 如: AF-A1B2-C3D4
}
```

```nginx
# API反向代理
location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
```

## 风险评估与应对

### 风险1：单点故障
- **风险**：服务器宕机导致服务中断
- **应对**：
  - 定期备份数据库
  - 准备快速恢复脚本
  - 考虑多服务器部署（后期）

### 风险2：ID冲突
- **风险**：生成的ID重复
- **应对**：
  - 服务端冲突检测
  - 冲突时自动重试
  - 使用足够大的ID空间

### 风险3：垃圾内容
- **风险**：匿名分享可能被滥用
- **应对**：
  - 添加简单频率限制
  - 准备举报机制（后期）
  - 定期清理过期数据

### 风险4：数据丢失
- **风险**：数据库损坏或误删除
- **应对**：
  - 每日自动备份
  - 异地备份（可选）
  - GitHub同步备份（可选）
