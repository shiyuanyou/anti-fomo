# Anti-FOMO v3.3 需求文档 (PRD-Lite)

## 核心目标

**目标1：实现前后端完全分离架构**
- 前端为纯静态资源，由Nginx直接托管
- 后端为独立FastAPI服务，仅提供API接口
- 前后端通过HTTP协议通信，物理隔离

**目标2：实现用户配置分享功能**
- 用户可在云端版创建自定义资产配置并命名
- 生成可分享的短链接（如 `https://anti-fomo.yoyoostone.cn/share/AF-A1B2-C3D4`）
- 其他用户可通过链接查看配置并一键复制到本地

## 详细需求

### 1. 模板库重构
- **数据源迁移**：官方模板从内存硬编码改为SQLite数据库存储
- **API端点**：`GET /api/templates` 返回所有官方模板列表
- **维护方式**：后续新增/修改模板只需操作数据库，无需改动代码或重新部署前端
- **数据量**：预估50-100个模板，SQLite完全胜任

### 2. 用户分享功能
- **创建分享**：用户在云端版点击"分享配置"，输入名称和作者，POST到 `/api/shares`
- **分享链接**：后端返回短ID，前端跳转至 `/share/:id` 独立页面
- **查看分享**：任何人可访问分享页，查看配置详情（资产饼图、权重、描述）
- **复制配置**：分享页提供"使用此配置"按钮，点击后将配置写入浏览器localStorage并跳转至编辑页
- **数据存储**：分享数据持久化到SQLite的`shares`表（作者、配置JSON、创建时间）

### 3. 双模式配置持久化（保持不变）
- **本地模式**：前端调用 `/api/assets` 和 `/api/save`，读写服务器本地的 `config.asset.yaml`
- **云端模式**：前端仅操作浏览器localStorage，分享功能通过 `/api/shares` 接口

### 4. 前端独立部署
- **构建产物**：`npm run build:cloud` 生成纯静态 `web/dist` 目录
- **部署方式**：将 `web/dist` 上传至服务器，Nginx直接serve静态文件
- **API调用**：前端使用完整域名 `fetch('https://anti-fomo.yoyoostone.cn/api/...')`
- **CORS支持**：FastAPI后端需配置跨域，允许前端域名访问

### 5. 本地开发体验
- **Vite Proxy**：本地开发时，前端 `/api` 请求代理到 `http://localhost:8000`
- **开发命令**：
  - 后端：`uvicorn api.main:app --reload`（监听8000端口）
  - 前端：`npm run dev`（监听5173端口，自动代理API）

## MVP边界（明确不做）

- ❌ 用户注册/登录系统（分享页为公开访问）
- ❌ 点赞、收藏、评论等社交功能（v3.4+再考虑）
- ❌ 分享数据分析、排行榜
- ❌ AI分析功能（推迟到v3.5，避免API Key泄露风险）
- ❌ 数据库管理后台（v3.3阶段使用SQL命令或脚本维护模板）
